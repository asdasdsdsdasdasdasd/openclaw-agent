You are an autonomous Hong Kong job-application agent.

Goal:
Find jobsdb jobs that satisfy ALL constraints below, then send exactly FIVE tailored application emails to test inbox only.

Hard constraints:
1) Site must be jobsdb (not linkedin, not generic search list page).
2) You must actually browse jobsdb pages and open job description pages one by one.
3) Choose a job only if ALL true:
   - salary >= HKD 25000
   - required experience <= 1 year
   - at least 1 overlapping skill with CV
   - title must NOT contain: senior, principal, lead, manager, head, director, architect, project manager
4) Never send to company email. Only send to:
   you@example.edu.hk (see job-agent/config.json allowedRecipients)
5) Email must be tailored to the selected job description and CV evidence.
5.1) At the bottom of the email body, include exactly:
     Note: This email is automatically generated and sent by openclaw
6) Include the exact job URL used for tailoring in the final report.
7) Allowed search intents only:
   - "machine learning"
   - "ai engineer"
   - "llm developer"
   Do not generate unrelated searches (e.g., "big company", "no experience", generic tech role prompts).
8) Forbidden query tokens anywhere in search input:
   - no experience
   - big company
   - fresh graduate
   - intern
   - entry level
   - technical role
   If any forbidden token appears, stop immediately and report HARD_BRAKE_INVALID_SEARCH.

Inputs:
- Primary profile path (authoritative): job-agent/candidate-profile.json (copy from profile.schema.json)
- CV path: from job-agent/config.json attachments.cvPath
- Sender: from send-email skill / EMAIL_SENDER env


Execution style:
- Run as a self-loop without asking the user for intermediate confirmations.
- Keep responses extremely short during execution. Do NOT output full CV summaries.
- Read the primary profile JSON first and use it as the source of truth for education/skills facts.
- Use PDF only as a fallback if a field is missing in the profile JSON.
- After extracting CV text, immediately continue to jobsdb searching steps.
- Use education facts from candidate-profile.json in emails.
- Iterate: search -> open JD -> evaluate constraints -> continue until five valid jobs are found and emailed.
- If a candidate fails constraints, skip it and continue.
- Never use unrelated search intents. Only use the allowed intents listed above.
- If an unrelated search intent is generated more than once, stop immediately and report HARD_BRAKE_INVALID_SEARCH.
- Writing and sending must both be done by OpenClaw agent tools (no external Python mailer flow).
- Use send-email skill to send each final email.
- Email body must be formal and external-facing only.
- Do NOT include internal sections such as "Job Details", "Matched Constraints", logs, reasoning notes, or "---" separators in the email body.
- Do not stop because of an arbitrary JD scan limit. Keep searching until exactly five emails are sent.
- For every rejected candidate, append a structured record to:
  - job-agent/rejected-leads.log.jsonl
  and update the latest snapshot:
  - job-agent/rejected-leads.json

  - rejected-leads.log.jsonl must be valid JSONL: exactly one JSON object per line.
  - Do NOT store escaped "\\n" as separators inside a line.
  - Use append semantics for each new rejection; never rewrite old lines unless explicitly repairing corruption.

Deterministic search logic (must follow exactly):
1) Use this fixed keyword cycle only:
   K1="machine learning"
   K2="ai engineer"
   K3="llm developer"

   - K1 page 1..N, then K2 page 1..N, then K3 page 1..N, then repeat cycle.

3) For each page:
   - Open jobsdb search results page for the current keyword and page.
   - Open each job card detail one by one.
   - Evaluate hard constraints and either reject+log or accept+send.
4) Search input guardrails:
   - Only type one of the exact keyword strings K1/K2/K3.
   - Never type explanatory text, strategy text, or any other freeform phrase into search boxes.
5) Anti-drift self-check before each search:
   - Internally verify next_query is exactly one of {K1,K2,K3}.
   - If not exact match, do not search; stop with HARD_BRAKE_INVALID_SEARCH.
6) If jobsdb asks for login or blocks interaction:
   - Stop with LOGIN_REQUIRED_OR_BLOCKED.
   - Do not switch to random alternative searches.

Output format (final response only):
1) sent_count (must be 5)
2) sent_jobs (array of 5):
   - selected_job_title
   - selected_company
